Theming Engine
CSS custom properties drive all visual output
Core
Every color, radius, shadow, font, and spacing value is defined as a var(). Changing data-theme on <html> swaps the entire look — zero re-renders, zero framework overhead. No React context, no styled-components, no runtime JS for styling.
Component Architecture
Decoupled, composable, framework-agnostic
16 components
Built with semantic HTML and atomic CSS classes. Components accept modifier classes (.btn-primary, .btn-lg, .card-header) instead of prop drilling. This pattern translates directly to React, Vue, or any component framework. Each component is theme-aware by default — no per-component theme code needed.

Overview

This design system uses CSS custom properties for theming. The entire UI adapts when you switch themes — no JavaScript re-rendering required. Components are built with semantic HTML and atomic CSS classes, making them portable to any framework.
Architecture

theme.css — All visual tokens as CSS custom properties
components.css — Component styles using var() references
base.css — Layout, reset, typography

Themes are applied by setting data-theme="corporate|midnight|sunset" on <html>.
Live Demo

Switch between the three themes in the header to see every component restyle instantly. The sidebar, cards, buttons, tables, badges, and form elements all respond without any framework reload.
Extending

To add a new theme, create a [data-theme="name"] block in theme.css and override the --color-* and --shadow-* variables. Every component picks up the new theme automatically.
